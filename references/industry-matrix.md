# 行业 × 职级决策矩阵

> 本文是 SKILL.md 中行业矩阵的完整扩展。包含全部 15 个行业的详细策略、改写倾斜规则、以及未覆盖行业的推导方法。

## 行业 × 职级全量策略

| 目标行业 | 全级别通用关注重点 | 实习/低年级 | 校招/应届 | 社招/跳槽 |
|----------|-------------------|------------|----------|----------|
| **互联网/科技** | 项目影响力、技术关键词、GitHub、量化产出 | 项目经历碾压（有完整项目 > 课程成绩）；GitHub 是关键证据 | 基础扎实 + 1-2 个深度项目；竞赛加分 | 业务影响力和量化产出；系统设计能力 |
| **金融/银行** | 学历资质、证书（CFA/FRM）、合规性、稳定性 | GPA + 学校 + 相关比赛 | 证书 + 实习 + 行业认知深度 | 交易额/项目规模/客户层级、合规意识 |
| **外企/咨询** | 英文表达、领导力、结构思维、教育含金量 | 国际经历、英语能力、课外领导力 | 咨询实习 + 案例能力 + 行业认知 | 项目规模、客户层级、跨文化协作 |
| **国企/体制内** | 政治面貌、学历层次、专业对口度、证书资质 | 学校层次 + 专业对口 + 学生干部经历 | 学历（硕士优先）+ 专业证书 + 相关实习 | 职称/资质、项目体量、稳定性 |
| **产品/运营** | 数据驱动（DAU/留存/GMV）、从 0 到 1 的经历 | 校内实践、社团活动、自建小产品 | 实习经历、数据思维、从 0 到 1 的案例 | 业务指标（DAU/留存/GMV）、团队管理 |
| **设计/创意** | 作品集链接、审美风格、工具链、项目类型跨度 | 作品集链接放最前面 | 实习经历 + 工具链广度 + 设计方法论 | 主导项目的影响力、设计系统构建能力 |
| **教育/培训** | 学历层次、教学成果数据、资格证书 | 专业对口 + 家教/助教经历 | 教师资格证 + 实习学校 + 公开课经历 | 教学成果量化（提分率/升学率）、课程研发能力 |
| **医疗/医药** | 学历资质、执业证书、临床/科研数据 | 专业排名 + 实验室经历 | 规培/实习医院 + 论文发表 | 临床案例量/手术量、科研项目经费、学术影响力 |
| **法律** | 学历（五院四系优先）、法律职业资格证、实习律所层级 | 模拟法庭 + 法律诊所 + 论文 | 律所实习 + 司法考试 + 业务领域聚焦 | 项目金额/标的额、客户层级、独立办案能力 |
| **建筑/工程** | 注册证书、项目体量、设计/施工管理能力 | 软件技能（CAD/BIM/SketchUp）+ 课程设计 | 设计院/工地实习 + 参与项目规模 | 持证等级、主持项目体量、造价/工期管控 |
| **媒体/新闻** | 作品集、传播数据、选题能力 | 校媒经验 + 个人自媒体 + 剪辑工具 | 媒体实习 + 爆款作品 + 平台运营 | 代表作阅读量/播放量、行业资源、深度报道能力 |
| **娱乐/演艺** | 代表作品、全网影响力数据（播放量/粉丝数/榜单排名）、荣誉奖项、演出经历、社交平台账号（微博/流媒体/短视频） | 作品集链接（B站/网易云/抖音）放头部；翻唱/翻跳/自媒体数据为王；展示多曲风/多风格跨度 | 专业能力证明（院校/师承/比赛成绩）+ 代表作品 + 传播数据；强调科班背景与赛事成绩 | 全网影响力数据（播放量/粉丝数/榜单登顶次数）、荣誉奖项、演出/巡演规模、商业合作层级、媒体曝光（央视/头部综艺） |
| **零售/电商** | GMV/SKU/转化率等运营数据、平台规则熟悉度 | 校内摆摊/闲鱼/微店等实践 | 电商实习 + 平台工具（千牛/巨量千川）+ 活动策划 | 店铺GMV体量、团队管理、供应链资源 |
| **游戏** | 项目经历（上线项目 > Demo）、引擎熟练度、品类理解 | 独立作品/Demo + 游戏Jam参与 | 实习项目 + 引擎专精 + 品类分析 | 上线项目数据（DAU/留存/付费）、管线经验 |
| **政府/非营利** | 政治面貌、公文写作、项目申报与管理能力；选调生需标注选调类别（定向/非定向）；公务员岗标注职务职级 | 学生干部 + 志愿服务 + 入党情况 + 基层服务项目（西部计划/三支一扶/特岗教师） | 基层实习 + 政策理解 + 组织协调 + 申论/行测成绩（如进入面试阶段可标注以体现笔试竞争力） | 项目管理经验、政策研究能力、跨部门协调；公务员岗标注职务职级与主管业务领域 |

## 矩阵未覆盖行业的通用策略

如果用户目标行业不在上述矩阵中，按以下三步骤现场推导：

1. **识别行业核心凭证**：这个行业用什么证明能力？（证书？作品集？项目数据？学历？从业年限？）——把它作为简历的核心卖点
2. **对标最相近的矩阵行**：从已有矩阵中找一个逻辑最接近的行业作为参考模板（如「保险」对标金融、「广告」对标设计/创意、「物流」对标零售/电商）
3. **向用户确认**：用一句话确认推导结果，格式：`你的行业不在我的预置矩阵中，我判断它比较接近「XX」行业的逻辑——核心卖点应该是 [A] 和 [B]。你看这个方向对吗？`

## 行业差异化结构

5 个差异最大的行业的模块布局（其余行业参考最相近行的逻辑）：

| 行业 | 结构特征 | 说明 |
|------|---------|------|
| **设计/创意** | 头部加作品集链接 → 项目经历（60%） → 技能（工具链 × 创意方向） → 工作经历 → 教育（紧凑） | 作品集是核心证据，占据视觉最高优先级 |
| **金融/银行** | 头部 → 教育（展开：学历+证书+GPA） → 工作/实习经历（核心） → 项目经历 → 技能 | 学历和证书是 ATS 第一关；量化精确到金额/百分比 |
| **法律** | 头部 → 教育（五院四系标注） → 职业资格（单独一栏：法律职业资格证/执业证号） → 实习律所经历 → 项目/案例 → 学术发表 | 职业资格证必须独立成栏（仅次于教育）；案例用标的额/案件量量化 |
| **教育/培训** | 头部 → 教学成果速览（可量化指标一览） → 工作经历（教学/管理） → 资格证书 → 项目经历（课程研发/公开课） → 教育背景 | 教学成果提到最前（提分率/升学率/带班规模）；资格证书独立成栏 |
| **娱乐/演艺** | 头部（含社交/作品平台链接） → 联系方式 → 个人简介/定位 → 影响力数据（播放量/粉丝数/奖项数） → 代表作品 → 荣誉奖项 → 工作/演出经历 → 教育背景 | 影响力数据是核心证据（ATS 与 HR 通用）；代表作品和奖项独立成栏，避免混入工作经历中 |

## 行业适应的改写倾斜

重写项目描述时，根据目标行业调整侧重点：

- **技术岗**：强调架构决策、技术选型理由、性能/安全考量
- **产品岗**：强调用户洞察、数据验证、迭代方向
- **设计岗**：强调设计系统构建、组件化思维、跨场景适配。**AI+设计混合岗**（用户工具链同时包含设计工具与 AI 工具如 ChatGPT/Gemini/Midjourney/即梦）：将 AI 工具链整合作为核心差异化卖点——不写成"用 AI 辅助设计"，而是"AI 驱动的创意工作流"——突出 AI 用于创意构思/素材生成/效率提升，传统设计工具用于精细排版/字效/合成的双层工作流
- **金融岗**：强调合规意识、风险控制、项目规模与精确数字
- **外企/咨询**：强调跨文化协作、领导力、结构思维（英文简历优先用 STAR 框架）
- **国企/体制内**：强调项目体量、稳定性、专业资质与学历背景
- **教育岗**：强调教学成果量化（提分率/通过率/升学率）、课程设计能力、学生评价
- **医疗岗**：强调临床/科研数据、合规与伦理、学术产出
- **法律岗**：强调案件量/标的额、胜诉率（如适用）、业务领域专精、文书写作能力
- **建筑/工程岗**：强调项目体量（面积/造价）、持证等级、BIM/数字化能力
- **媒体/新闻岗**：强调传播数据（阅读量/播放量/互动率）、选题方法论、跨平台运营
- **娱乐/演艺岗**：强调全网影响力数据（播放量/粉丝数/榜单排名/热搜次数）、代表作品是核心证据（而非工作经历）、荣誉奖项独立成栏、多曲风/多语言/跨领域为差异化亮点。代表作用「现象级」「出圈」「破圈」等叙事；量化到"专辑销量/演唱会人次/音源播放量/综艺观看量"。社交平台作品链接（B站/网易云/抖音/微博）放头部联系方式行，与电话/邮箱同行
- **零售/电商岗**：强调 GMV/SKU/ROI 等运营指标、供应链与库存管理、平台规则熟悉度
- **游戏岗**：强调上线数据（DAU/留存/LTV）、品类方法论、管线协作经验
- **政府/非营利岗**：强调公文写作能力（讲话稿/调研报告/政策文件类型）、组织协调案例（会议/活动/项目管理）、政策研究与建议采纳、基层服务时长与考核结果
- **未覆盖行业**：参考通用策略的三段式推导，从行业核心凭证反推改写重点

## 设计岗作品集链接处理

- 作品集链接必须放在头部联系方式行，与电话/邮箱同行或紧邻下一行
- 支持链接类型：站酷、Behance、Dribbble、小红书、百度网盘、飞书文档、Notion、Google Drive、GitHub Pages 等
- **网盘链接特殊处理**：百度网盘/阿里云盘/夸克网盘等链接，保留完整 URL（含提取码参数），禁止把提取码从 URL 中拆出来单独写
- 禁止在头部直接嵌入图片二维码（ATS 无法解析）

## 不可放的内容（全行业通用）

照片（除非地区招聘习惯要求）、民族、政治面貌（除国企明确要求外）、身高体重、身份证号、自我评价段落（用项目经历证明能力，不要用形容词描述性格）

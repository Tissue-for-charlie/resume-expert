# 项目描述重写规则

> 本文是 SKILL.md 模式 A Step 4 的完整重写规则扩展。包含重写对照表、行业倾斜、英文简历 STAR 框架和动作动词库。

## 重写原则

用户提供的原始描述往往是**规格说明书风格**（"该系统包括 A、B、C 功能……"）或**任务清单风格**（"我负责了 XX；我还负责了 YY"）。必须重写为**个人实践叙事**：

| 原来 | 改为 |
|------|------|
| "负责 XX 模块开发" | "基于 YY 技术，独立完成 XX 模块，解决 ZZ 问题" |
| 功能罗列（A、B、C、D……） | 架构式描述：整体结构 → 各模块职责 → 协作链路 |
| 无数据 | 补充量化结果：响应时间、准确率、团队规模、迭代轮次（向用户确认后加入） |
| 被动语态 / "该系统" | 主动语态——"设计/实现/优化/构建/重构"开头 |
| 技术栈堆砌 | 融入叙事：在描述上下文中自然出现技术名词，而不是单独列一行 |

## 英文简历专项规则

**仅当用户选择英文或双语简历时生效。**

### STAR 框架

STAR = Situation（情境）→ Task（任务）→ Action（行动）→ Result（结果）

**示例对比**：

> ❌ Responsible for developing the payment module. Used Spring Boot and MySQL. Fixed several bugs.

> ✅ Migrated the payment module from a monolithic architecture to microservices (Spring Boot + MySQL), reducing transaction latency by 40% and eliminating 3 recurring production incidents.

### 中英文规则差异

| 规则 | 中文简历 | 英文简历 |
|------|----------|----------|
| 描述框架 | 个人实践叙事 | STAR 框架 |
| 动词时态 | 不适用 | 当前职位用现在时（"Designs..."），过往经历用过去时（"Designed..."） |
| 页数 | 实习/校招 1 页，社招可 2 页 | 严格 1 页（10 年以下经验一律 1 页） |
| 动作动词 | "负责/主导/实现/优化" | "Designed/Implemented/Optimized/Led/Architected" |
| 人称 | 省略主语 | 省略 "I" / "My"，直接动词开头 |
| 联系方式 | 电话 + 邮箱 | 电话 + 邮箱 + LinkedIn（必须） |

### 常用英文动作动词（按职能分类）

| 职能 | 动词 |
|------|------|
| 技术 | Engineered, Architected, Implemented, Optimized, Refactored, Migrated, Automated |
| 管理 | Led, Directed, Coordinated, Mentored, Spearheaded, Orchestrated |
| 产品/数据 | Identified, Analyzed, Drove, Launched, Scaled, Instrumented |
| 设计 | Designed, Prototyped, Iterated, Branded, Revamped |

### 弱动词替换对照

| ❌ 弱动词 | ✅ 替换 |
|-----------|--------|
| Worked on | Developed, Built, Implemented |
| Helped with | Contributed to, Co-developed |
| Was part of | Collaborated on, Joined |
| Did | Performed, Executed, Completed |
| Made | Created, Developed, Produced |
| Got | Achieved, Attained, Secured |

### 常见错误

1. **用代词 "I" / "My"** → 省略主语，直接动词开头
2. **被动语态** → 主动语态
3. **动词重复** → 同段 bullet 用不同动词开头
4. **中文习惯直译** → "负责"不译为 "Responsible for"，直接动词开头

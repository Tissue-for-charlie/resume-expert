# 评分系统 Golden Test Set（金标准测试集）

> 本文是 [`scoring-system.md`](scoring-system.md) 的配套校准文件。包含 **75 份**经过人工专家评分的标准简历，覆盖 **15 个行业 × 5 个分数段**，用于验证 AI 评分的一致性。

## 使用方式

**对于 AI**：在评审用户简历（模式 B Step 2）前，建议先速读本文中与目标行业/职级最接近的 2-3 个测试用例，建立分数段基准感，再对用户简历打分。

**对于人类贡献者**：新增行业或修改评分规则后，应同步更新本文——至少新增该行业的 5 个代表性测试用例（每个分数段 1 个）。

**验证方法**：将一份 golden test 简历粘贴到对话中说"帮我看一下这份简历"，对比 AI 评分与专家评分的差异。允许 ±5 分的波动（评分本质上存在主观性），但维度间相对大小和总分分数段应保持一致。

## 🌐 Web 搜索增强说明

**部分测试用例标注 `🌐 Web 搜索校准`**，表示该 case 的简历内容和评分基准参考了 2025-2026 年互联网公开的招聘数据（JD 关键词、行业薪资报告、简历范文）。这些 cases 的基准中位数已根据搜索到的市场报告（RGF 薪酬观察 2026、智联春招周报 2026、脉脉高聘 2026 春招洞察等）进行了校准。

**AI 评分时的搜索增强**（可选）：当 WebSearch 工具可用时，在评分前搜索目标行业的当前市场数据——这有助于理解当前热门技术栈/证书/竞争环境，使评分更贴近当下市场现实。

## AI 评分前校准流程

在评审用户简历（模式 B Step 2）之前，建议执行以下自检：

1. **找锚点**：从本测试集中找到与目标行业/职级最接近的 2-3 个用例（同行业优先，相邻行业其次），速读其评审结果，建立"这个分数段的简历长什么样"的基准感
2. **对齐权重**：确认目标行业的四维权重映射（见 [`scoring-system.md`](scoring-system.md)），不要用默认权重去评特殊行业（如设计岗排版权重 30、金融岗关键词权重 25）
3. **评分后自检**：打完分后，将用户简历与同分数段的 golden test 简历做一次对比——它们的质量水平是否确实相近？
4. **允许 ±5 波动**：评分存在主观成分，同一份简历在不同对话中 ±5 分属于可接受范围。但如果波动超过 5 分，说明评分标准发生了漂移，应重新校准
5. **🌐 Web 搜索增强（可选）**：当 WebSearch 工具可用时，建议在评分前搜索目标行业的当前市场数据，以获取最新的关键词趋势和竞争环境信息。搜索策略：
   - 搜索 query 模板 1：`"{行业} {职级} 简历 面试经验 2026"`
   - 搜索 query 模板 2：`"{行业} 招聘 JD 核心关键词 2025 2026"`
   - 搜索 query 模板 3：`"{行业} 薪资 竞争 2026"`
   - 将搜索结果中的热门技术栈/证书/量化指标用于校准**关键词覆盖**和**差异化定位**两个维度的评分
   - 如果搜索到行业薪资报告/竞争分析，可用于修正基准中位数估计
   - **不可用时回退**：WebSearch 不可用时（如离线/非 Claude Code 平台），完全依赖本文件中的静态 golden test 进行校准即可
